---
title: "Reset MySQL root password"
description: "How to reset the MariaDB root password when you are locked out of MySQL on your Frappe server."
---

<Warning>
  This procedure is for emergency use only — use it when you are locked out of MySQL and cannot authenticate as root through normal means. It requires stopping the MySQL service, which will interrupt any running Frappe sites.
</Warning>

If you cannot connect to MySQL as root (for example, after a failed `mysql_secure_installation` or a forgotten password), follow these steps to reset the root password.

<Steps>
  <Step title="Stop the MySQL service">
    Stop the running MySQL service and ensure no `mysqld` processes remain:

    ```bash
    sudo systemctl stop mysql
    killall mysqld
    ```
  </Step>

  <Step title="Create an init file with the new password">
    Create a temporary init file that MySQL will execute on startup to set the new password. Replace `[username]` with your Linux username and `NewPassword` with your chosen password:

    ```bash
    sudo nano /home/[username]/mysql-init
    ```

    Add the following line to the file:

    ```sql
    ALTER USER 'root'@'localhost' IDENTIFIED BY 'NewPassword';
    ```

    Save and close the file (Ctrl+X, Y, Enter).
  </Step>

  <Step title="Start MySQL with the init file">
    Start `mysqld` using the init file to apply the password change, then start the service normally:

    ```bash
    sudo mysqld --init-file=/home/[username]/mysql-init &
    sudo service mysql start
    ```
  </Step>

  <Step title="Start MySQL in safe mode (alternative method)">
    If the init file method does not work, start MySQL in safe mode with grant tables disabled:

    ```bash
    sudo mysqld_safe --skip-grant-tables --skip-networking &
    ```

    <Note>
      The `--skip-networking` flag prevents remote connections while in safe mode, limiting access to the local socket only.
    </Note>
  </Step>

  <Step title="Connect to MySQL as root">
    Open a new terminal and connect to the running MySQL instance without a password:

    ```bash
    sudo mysql -u root
    ```
  </Step>

  <Step title="Update the root password">
    Run the following SQL commands to set a new root password and reload the privilege tables:

    ```sql
    USE mysql;
    UPDATE user SET authentication_string=PASSWORD('new_password') WHERE User='root';
    FLUSH PRIVILEGES;
    ```

    Replace `new_password` with a strong password of your choice. Then exit the MySQL prompt:

    ```sql
    EXIT;
    ```
  </Step>

  <Step title="Stop the safe mode process">
    Kill the `mysqld_safe` process that is running in the background:

    ```bash
    sudo killall mysqld_safe
    ```
  </Step>

  <Step title="Start MySQL normally">
    Start the MySQL service using the standard method:

    ```bash
    sudo systemctl start mysql
    ```

    Verify you can now connect with the new password:

    ```bash
    sudo mysql -u root -p
    ```
  </Step>
</Steps>

<Tip>
  After resetting the password, delete the temporary init file to avoid leaving credentials on disk:

  ```bash
  sudo rm /home/[username]/mysql-init
  ```
</Tip>
